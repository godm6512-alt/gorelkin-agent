# INSTALL.md — инструкция для Claude Code

> **Это служебный файл.** Его читает Claude Code, когда ученик просит установить архитектуру Агента. После завершения установки этот файл удаляется.

---

## Кто ты сейчас

Ты — Claude Code, открытый в папке проекта ученика курса «Архитектор нейросотрудников». Ученик только что попросил установить архитектуру Агента из шаблона `Ntmib/jarvis-architect`.

К моменту чтения этого файла шаблон уже скачан (через `npx degit Ntmib/jarvis-architect .` или прямой `git clone`). Ты находишься в папке ученика, рядом с папкой `core/` где лежат шаблоны.

## Что нужно сделать (по шагам)

### Шаг 1. Поздоровайся и объясни что будешь делать (1 минута)

Скажи примерно так (адаптируй, не копируй дословно):

> «Привет. Я установил шаблон вашего Агента. Сейчас задам 10 коротких вопросов про вас и ваши цели — чтобы заполнить файлы под вас, а не оставлять пустые. Это займёт 5–7 минут. Готовы?»

Дождись ответа. Если ученик не готов сейчас — скажи что инструкция остаётся, можно вернуться позже.

### Шаг 2. Прочитай все 4 шаблона в `core/`

Файлы:
- `core/CLAUDE.md` — правила работы Агента в проекте
- `core/SOUL.md` — личность Агента
- `core/MEMORY.md` — блокнот фактов
- `core/GOALS.md` — цели

В каждом файле есть плейсхолдеры в фигурных скобках вроде `{{NAME}}`, `{{ROLE}}`, `{{COMMUNICATION_STYLE}}`. Их нужно заменить ответами ученика.

### Шаг 3. Проведи интервью

Задавай **по одному вопросу за раз**, дожидайся ответа. Не перечисляй сразу все 10 — это давит.

**Вопросы (порядок важен — от простого к глубокому):**

1. **«Как мне к вам обращаться?»** → имя ученика. Сохрани в переменную `{{NAME}}`.
2. **«Чем вы сейчас занимаетесь — основная роль или профессия?»** → роль. `{{ROLE}}`.
3. **«Какие 1–3 главных проекта у вас в работе прямо сейчас?»** → список проектов. `{{ACTIVE_PROJECTS}}`. Если ученик затрудняется — спроси «над чем хотите чтобы Агент помогал в первую очередь?».
4. **«Какую главную цель хотите достичь за 3 месяца?»** → `{{Q_GOAL}}`. Можно несколько — но одну главную выдели.
5. **«На “Вы” или на “ты” к вам обращаться?»** → `{{ADDRESS_FORM}}`. По умолчанию — «Вы».
6. **«Какой стиль ответов комфортен — коротко по делу, или подробно с примерами?»** → `{{ANSWER_STYLE}}`.
7. **«Что вас бесит в общении ИИ-ассистентов? Скажите 2–3 вещи».** → `{{IRRITATIONS}}`. Это важный вопрос — отвечает на «чего НЕ делать».
8. **«Какие три слова описывают, какого Агента вы хотите рядом с собой?»** → `{{AGENT_TRAITS}}`. Например: «спокойный, прямой, конкретный».
9. **«На каком языке общаться?»** → `{{LANGUAGE}}`. По умолчанию — русский.
10. **«Что Агент может делать сам без вашего разрешения, а что только с вашим “да”?»** → `{{AUTONOMY_RULES}}`. Если ученик не знает — предложи безопасный дефолт: *«Сам — читать файлы, искать в интернете, писать заметки. С вашим “да” — удалять файлы, отправлять сообщения, делать платежи»*.

**Не задавай больше вопросов, даже если кажется что мало данных.** Файлы можно дополнять потом — это нормально.

### Шаг 4. Покажи что получилось (важно)

Прежде чем что-то перемещать или удалять — заполни плейсхолдеры в файлах из `core/` и покажи ученику в чате превью каждого:

> «Вот что получилось — посмотрите. Если где-то поправить — скажите, я перепишу.»

Покажи поочерёдно содержимое:
1. `SOUL.md` — самое личное, ученику важно увидеть «это про меня»
2. `CLAUDE.md` — правила работы
3. `MEMORY.md` — что уже знает Агент
4. `GOALS.md` — цели

Дождись подтверждения «всё ок» или правок. Если правки — внеси их и снова покажи.

### Шаг 5. Перенеси файлы в корень и почисти служебное

После подтверждения ученика выполни:

```bash
mv core/CLAUDE.md ./CLAUDE.md
mv core/SOUL.md ./SOUL.md
mv core/MEMORY.md ./MEMORY.md
mv core/GOALS.md ./GOALS.md
rm -rf core/
rm -rf examples/
rm -rf docs/
rm -rf dna-map/
rm INSTALL.md
rm README.md
```

Объясни ученику простыми словами: *«Перенёс ваши заполненные файлы из служебной папки `core/` в корень проекта и удалил всё лишнее, что было нужно только для установки».*

> **Важно:** папку `dna-map/` мы тоже удаляем. Файлы MISSION/PROJECTS/PREFERENCES/LEARNED — это карта для будущего, ученик создаст их когда понадобится. В шаблоне они пустые и хранить их в папке нет смысла.

### Шаг 6. Первый коммит и push

Объясни что делаешь. Запускай ОДИН коммит, не дроби:

```bash
git add -A
git commit -m "Архитектура Агента собрана"
git push
```

Если ученик ещё не подключил GitHub — спроси и помоги (через Урок 3 у него уже должен быть `git remote`).

### Шаг 7. Финал

Скажи примерно:

> «Готово. У вас теперь 4 файла, которые делают вашего Агента — Агентом, а не чатом. Откройте `SOUL.md` — это первая страница его паспорта. В следующем уроке добавим команду подчинённых.»

## Чего НЕ делать

- **Не перемещай файлы из `core/` ДО того как ученик подтвердил содержимое.**
- **Не делай больше 10 вопросов** — это интервью, не допрос.
- **Не заполняй плейсхолдеры догадками.** Если ответ ученика непонятен — переспроси.
- **Не пропускай Шаг 4** (показ превью). Это критическая точка контроля.
- **Не удаляй `.git/`** или `.gitignore` — они нужны.
- **Не сохраняй ответы интервью в отдельный файл** (типа `interview-answers.md`). Они должны попасть только в шаблоны.

## Если что-то пошло не так

- **Ученик хочет начать заново** → удали все заполненные файлы, восстанови шаблоны из `core/`, начни с Шага 1.
- **Ученик не знает что отвечать на вопрос** → дай два-три варианта на выбор. Например для «стиля ответов»: «коротко и по делу / средне с пояснением / подробно с примерами».
- **Ученик хочет посмотреть пример заполненного файла** → покажи `examples/jarvis-example/SOUL.md` (он есть в шаблоне).
- **Если ты Claude Code в режиме Ask Before Edit** — для команд `mv`, `rm`, `git` нужно подтверждение. Объясняй ученику что собираешься сделать перед каждой такой командой.
