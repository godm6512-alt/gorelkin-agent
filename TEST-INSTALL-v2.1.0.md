# Тестирование v2.1.0 — инструкция для волонтёра

Эта версия пока не зарелижена. Цель — проверить установку на одном агенте перед общим выкатом.

## Что появится после обновления

- Команда `/connect` — VS Code через бота за минуту (GitHub OAuth → ссылка vscode.dev)
- Команда `/reauth` — переподключение Claude когда подписка отвалилась (без обращения в поддержку)
- Кнопка «Переподключить Claude» в /settings
- Сообщение «Готов к работе» после рестарта при reauth

## Что от тебя нужно

1. Запустить одну команду на своём сервере (см. ниже)
2. Проверить что бот ответил и работает как раньше
3. Написать в Telegram /connect — должна прийти GitHub-авторизация
4. После авторизации проверить что ссылка vscode.dev/tunnel/agent-XXXXXXXX открывается
5. Написать `@mcdenil`: «работает» или «не работает + что именно»

Если что-то сломается — у бота есть автоматический бэкап в `/home/agent/.agent/bot/.backup-*`, откатывается одной командой (см. в конце).

## Команда обновления

Подключись к серверу любым способом (старый туннель, SSH, что угодно). В терминал вставь:

```bash
curl -fsSL https://raw.githubusercontent.com/Ntmib/jarvis-architect/feature/connect-and-reauth/bot/update-bot.sh -o /tmp/update-bot.sh && BRANCH=feature/connect-and-reauth bash /tmp/update-bot.sh
```

Что произойдёт за 2-3 минуты:
- Скачаются новые файлы
- Бэкап старых файлов в `.backup-YYYYMMDD-HHMMSS/`
- Установится инфраструктура VS Code Tunnel (если её нет)
- Бот перезапустится

В конце увидишь `=== Update complete! Version: 2.1.0 ===`.

## Проверка после обновления

### 1. Бот отвечает
Напиши в Telegram любое сообщение — бот должен ответить как обычно.

### 2. /connect работает
Напиши `/connect`. Должно прийти:
- Сообщение «Запускаю VS Code подключение»
- Через 5-10 секунд — ссылка на `github.com/login/device` + код вида `XXXX-XXXX`
- Открой ссылку в браузере, введи код, выбери GitHub-аккаунт
- Через минуту в Telegram придёт «✔ VS Code подключение готово!» + ссылка `vscode.dev/tunnel/agent-XXXXXXXX`
- Открой ссылку — должен открыться VS Code с твоим сервером

### 3. /reauth работает (опциональная проверка)
Напиши `/reauth`. Должны прийти:
- 2 скриншота с инструкциями
- Кнопка «🔗 Авторизоваться в Claude»
- Открой ссылку, авторизуйся, скопируй адрес из адресной строки браузера
- Вставь адрес в бота
- Через 10-20 секунд: «✅ Готов к работе!»

Если боишься сломать рабочую авторизацию Claude — пропусти этот пункт, проверим вместе.

## Если что-то пошло не так

### Бот не отвечает после обновления
```bash
systemctl status agent-bot
journalctl -u agent-bot -n 50
```
Если видишь ошибку — пришли её админу.

### Откатиться на v2.0
В папке бота лежит автоматический бэкап. Откат:

```bash
cd /home/agent/.agent/bot
LATEST_BACKUP=$(ls -dt .backup-* | head -1)
cp -r "$LATEST_BACKUP"/* .
systemctl restart agent-bot
```

### Старый VS Code-туннель пропал
Если раньше пользовался туннелем от `code tunnel service install` (старый способ) — он будет остановлен. Новый туннель появится только после `/connect`. Это нормально.
