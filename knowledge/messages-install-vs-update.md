# Сообщения для учеников: установка с нуля vs обновление

Дата: 2026-07-05. Проверено по коду и на живых запросах.

## ВАЖНО: сломанные пути обновления (не советовать)
Проверено по main index.js (2026-07-05):
- Встроенный `/update` в боте (handler строки 1888-1899) запускает `update-bot.sh`
  С ДИСКА, свежий качает только `if (!existsSync)`. У v2.0-ученика на диске старый
  скрипт (без claude-oauth.js/env-write.js) → index.js v2.1.0 падает ERR_MODULE_NOT_FOUND
  → автооткат. Итог: `/update` в чате для скачка v2.0→v2.1.0 НЕ РАБОТАЕТ.
- Фолбэк `bash /home/agent/.agent/bot/update-bot.sh` = тот же старый on-disk скрипт = ломается.
- `knowledge/update-guide.md` сценарии A/B/C ведут по этим же сломанным путям — не линковать
  до починки.
Старый broadcast (с «напиши /update руками») я советовал ДО находки бага — он сломан.

ЕДИНСТВЕННЫЙ рабочий путь v2.0→v2.1.0: скачать свежий скрипт в /tmp и запустить (обходит
старый on-disk). Память при этом не трогается (см. ниже).

Ключевое различие:
- `/new` в боте = установка АГЕНТА С НУЛЯ в новую папку. Памяти ещё нет — терять нечего.
- update-bot.sh = ОБНОВЛЕНИЕ уже стоящего агента. Память (MEMORY.md, SOUL.md,
  GOALS.md, memory/, knowledge/, memory.db) НЕ трогается — проверено по путям.

Почему обновление безопасно для памяти (факты из кода bot/):
- Скрипт пишет только в `/home/agent/.agent/bot/` (код бота).
- Память в других папках: `/home/agent/workspace/` (ДНК + дневники + knowledge)
  и `/home/agent/.agent/memory.db` (индекс поиска). Скрипт их не касается.
- Единственный `rm -rf` — по временной папке в /tmp. Старый код бэкапится в `.backup-…`.
- Рестарт не запускает повторный bootstrap (guard: bootstrapDone = hasExistingFiles).
- update-bot.sh токен не использует, репо публичный → обновление ничего не требует, кроме доступа к терминалу.

## СООБЩЕНИЕ 1 — Поставить агента с нуля (у кого его ещё нет)

Ставим личного AI-агента впервые.

1. Откройте Telegram-бот @jarchitect_bot и отправьте команду /new
2. Бот пришлёт готовый промпт с вашей персональной ссылкой на архив
3. Откройте Claude Code на компьютере (или на сервере)
4. Скопируйте промпт целиком, вставьте в Claude Code, нажмите Enter
5. Дальше агент всё сделает сам: скачает архив, распакует, задаст 10 вопросов для настройки

Одна ссылка = до 3 скачиваний. Если бот ответит «доступ закрыт» — напишите мне, добавлю в список.

## СООБЩЕНИЕ 2 — Обновить агента до v2.1.0 (у кого он уже стоит, память сохранится)

🆕 Версия 2.1.0 — обновление бота-агента

Что нового:

/connect — VS Code через бота за минуту.
Пишешь /connect → бот даёт ссылку github.com/login/device и код → подтверждаешь →
бот присылает ссылку vscode.dev/tunnel/agent-XXXXXXXX. Открываешь в браузере или в
десктопном VS Code. SSH-клиент не нужен.

/reauth — переподключить Claude без поддержки.
Подписка Claude отвалилась → пишешь /reauth → авторизуешься на claude.ai → копируешь
URL обратно в бота → он сам перезапускается с новым токеном.

Плюс /version и /update.

⚙️ Как обновиться (память сохранится)

Открой терминал на сервере, где стоит агент, и выполни ОДНУ команду:

curl -fsSL https://raw.githubusercontent.com/Ntmib/jarvis-architect/main/bot/update-bot.sh -o /tmp/update-bot.sh && bash /tmp/update-bot.sh

Она скачивает свежий установщик, обновляет только код бота, делает бэкап и
перезапускает. Через 1-2 минуты появятся новые команды.

Терминал открывается по SSH или через VS Code. Если бот на VPS — запускай под root.

Почему именно так, а не /update в чате: у старых версий встроенный /update запускает
устаревший установщик и откатывается. Команда выше берёт актуальный — и ставится с
первого раза.

🛟 Что сохранится
SOUL.md, MEMORY.md, GOALS.md, дневники, база знаний, все настройки и токены.
Обновляется только код бота. При сбое — автооткат на прежнюю версию.

Важно (для нас, не в текст ученику): НЕ распаковывать бонусный архив поверх уже
установленного агента — шаблонные файлы затрут память. Для обновления — только
команда выше.
