# v2.1.0 — почему у людей не ставится обновление (root cause)

Дата разбора: 2026-07-05. Статус: причина подтверждена доказательством.

## Симптом
Ученики жмут «обновить» (через /update или через терминал) — обновление
не устанавливается: бот остаётся на v2.0.0, новых команд /connect, /reauth,
/version нет. Часто «Готово» не приходит либо приходит откат.

## Корневая причина (bootstrap-ловушка ESM)

1. Новый `bot/index.js` (v2.1.0) содержит БЕЗУСЛОВНЫЕ top-level ESM-импорты:
   - `import ... from "./lib/claude-oauth.js"` (строки 30-35)
   - `import { safeEnvWrite, safeEnvRemove } from "./lib/env-write.js"` (строка 36)

2. Старый `update-bot.sh`, который лежит на серверах v2.0-учеников
   (коммит `e3d2c71`), в списке файлов НЕ содержит этих двух модулей:
   `LIB_FILES="lib/db.js lib/embeddings.js lib/memory-indexer.js lib/memory-search.js"`
   Нет ни claude-oauth.js, ни env-write.js, ни images/, ни блока tunnel.

3. Старый скрипт качает НОВЫЙ index.js (он всегда в CORE_FILES), но не качает
   его новые зависимости.

4. Проверка `node --check index.js` в скрипте проходит — потому что `--check`
   валидирует ТОЛЬКО синтаксис и НЕ резолвит импорты. Доказано:
   `node --check` на файле с `import "./lib/env-write.js"` (файла нет) = PASS,
   а `node file.mjs` = `ERR_MODULE_NOT_FOUND`.

5. Скрипт устанавливает новый index.js, перезапускает бота → бот падает на
   старте с `ERR_MODULE_NOT_FOUND: ./lib/claude-oauth.js` → update-bot.sh
   ловит «бот не active» → **автооткат на v2.0**. Ученик снова на v2.0.

## Почему у волонтёра сработало, а у остальных нет
TEST-INSTALL-v2.1.0.md даёт ПРАВИЛЬНУЮ команду — сначала скачать СВЕЖИЙ
update-bot.sh в /tmp, потом запустить его (у свежего полный список файлов):
`curl ... update-bot.sh -o /tmp/update-bot.sh && bash /tmp/update-bot.sh`

А CHANGELOG-v2.1.0.md и knowledge/update-guide.md (сценарии A/B/C) велят
запускать СТАРЫЙ скрипт с диска / жать /update — оба гоняют старый список.

## Все документированные пути и их судьба
- Сценарий A/B — `/update` в боте. Хендлер v2.0 запускает старый on-disk
  update-bot.sh. Хендлер v2.1.0 качает свежий скрипт ТОЛЬКО если его нет
  (`if (!existsSync)`), иначе запускает существующий. Для скачка v2.0→v2.1.0
  = старый список = ЛОМАЕТСЯ.
- Сценарий C — `bash /home/agent/.agent/bot/update-bot.sh` = старый список = ЛОМАЕТСЯ.
  Либо у совсем старых установок файла нет → «Bot directory not found».
- TEST-INSTALL (curl свежий в /tmp → запуск) = РАБОТАЕТ.

## Две подгруппы пострадавших
1. update-bot.sh на диске = e3d2c71 (без новых lib) → обновление + откат.
2. Установка до появления update-bot.sh (до ~16 мая) → файла нет, сценарий C падает.

Важно: images/step5_*.png и templates/vscode-tunnel/ старый скрипт тоже не
качает — но это НЕ фатально (ломает /reauth-скриншоты и /connect, но не старт
бота). Фатальны именно claude-oauth.js + env-write.js.

## Немедленное лечение для застрявших (одна команда)
Скачать СВЕЖИЙ скрипт и запустить его — он тянет полный список:
```
curl -fsSL https://raw.githubusercontent.com/Ntmib/jarvis-architect/main/bot/update-bot.sh -o /tmp/update-bot.sh && bash /tmp/update-bot.sh
```

## Постоянный фикс (предложение, не выполнено)
1. update-bot.sh: в самом начале скачивать свежую версию себя в /tmp и
   re-exec (с guard-переменной от зацикливания) — тогда любой старый скрипт
   бутстрапится до актуального списка файлов.
2. update-bot.sh: после скачивания index.js — проверить, что ВСЕ его
   `import "./..."` реально скачаны, иначе abort ДО замены (не доводить до
   отката). `grep -oE 'from "\./[^"]+"' index.js`.
3. /update-хендлер: всегда качать свежий update-bot.sh в /tmp и запускать его
   (убрать `if (!existsSync)`). Помогает только тем, кто уже на v2.1.0+.
4. Доки (CHANGELOG, update-guide сценарий C) — заменить на self-bootstrap curl.
