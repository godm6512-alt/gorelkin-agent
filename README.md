# jarvis-architect

Шаблон архитектуры личного Агента в Claude Code. Часть курса **«Архитектор нейросотрудников»** Дмитрия Ледовских.

## Что это такое

Готовый набор файлов, из которых собирается ваш персональный Агент:

| Файл | Что внутри |
|---|---|
| `CLAUDE.md` | Правила работы Агента в этой папке. Claude Code читает первым. |
| `SOUL.md` | Конституция Агента: личность, тон, ценности, что любит / не любит. |
| `MEMORY.md` | Блокнот: факты о вас, проекты, ключевые решения. |
| `GOALS.md` | Цели на квартал и активные задачи. |
| `dna-map/` | Опциональные файлы (создаются по необходимости): MISSION, PROJECTS, PREFERENCES, LEARNED. |
| `.claude/skills/` | Команда подчинённых-скиллов (появятся в Уроке 5). |

Файлы пустые — они заполняются в процессе короткого интервью с Claude Code.

## Как установить (Урок 4)

В вашем Claude Code, открытом в папке `jarvis/`, скажите одну фразу:

> «Установи мне архитектуру Агента из github.com/Ntmib/jarvis-architect»

Дальше Claude сам:

1. Скачает шаблон в текущую папку через `npx degit`
2. Прочитает `INSTALL.md` — там пошаговая инструкция для него
3. Задаст вам 10–15 вопросов: имя, чем занимаетесь, цели, стиль общения, что бесит
4. Заполнит шаблоны на основе ответов
5. Перенесёт готовые файлы в корень папки и удалит служебное
6. Сделает первый коммит + push в ваш GitHub-репозиторий

Время от команды до готовой архитектуры — 10–12 минут.

## Если предпочитаете команду в терминале

```bash
npx degit Ntmib/jarvis-architect .
```

После этого откройте Claude Code в этой папке — он сам найдёт `INSTALL.md` и продолжит установку.

## Если что-то пошло не так

В папке `examples/` лежат заполненные примеры — подсмотрите, как должны выглядеть готовые файлы. Если интервью с Claude сбилось, можно скопировать пример и поправить под себя.
