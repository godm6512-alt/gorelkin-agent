
## Сессия (диагностика 401 демо-бота)
**Тема:** почему демо-бот теряет OAuth-токен после входа в claude через терминал

### Что установил
- Демо-бот = инстанс jarvis-architect под юзером `agent`, сервис `agent-bot`, env `/home/agent/.agent/.env`, credentials `/home/agent/.claude/.credentials.json`. НЕ на constructor-dmitrij (тут только личный .iia-Jarvis).
- Бот спавнит `claude` с HOME=/home/agent (index.js:682-684) → читает .credentials.json, CLI сам рефрешит access-токен, refresh-токен РОТИРУЕТСЯ.
- Root cause 401: одна подписка Claude Max = одна ротируемая цепочка refresh-токена. Вход `claude /login`/`setup-token` в терминале под тем же аккаунтом выдаёт новый токен и обесценивает тот, что держит бот → следующий спавн claude ловит 401. Два независимых инстанса на одной подписке дерутся за токен.
- У учеников не ломается: у каждого свой Anthropic-аккаунт (перекрытия нет). Ломается у Дмитрия, т.к. демо-бот и его терминал/личный Jarvis на одном аккаунте.
- bash_history (20.06) подтверждает: делал claude auth login + setup-token + правил ~/.iia/.env.
- Фикс: reauth через бота (/settings → Переподключить Claude) или ручной claude /login под agent + restart (knowledge/claude-reauth-manual.md). Профилактика: отдельный Anthropic-аккаунт для демо-бота ЛИБО не логиниться в claude под тем же аккаунтом.

### Безопасность
- В ~/.bash_history засветился GitHub-токен ghp_... в открытом виде (git remote set-url). Предложить Дмитрию отозвать.
